
# Hiring Guideline

## Hiring Standard

### fika.d team core values

**고객의 문제에 집착합니다.**

- 피카디는 고객의 불편함을 우리의 문제처럼 세심하게 살피고, 깊이 공감하며 해결책을 찾는 팀입니다.
- 피카디는 나의 추측보다는 고객이 실제로 겪는 경험과 상황을 판단의 기준으로 삼습니다.
- 고객의 피드백이 우리의 나침반입니다. 때로는 쓴소리일지라도, 그 안에 우리가 나아가야 할 방향과 진실이 담겨 있다고 믿습니다.

주도적으로 일합니다.

- 피카디의 팀원은 단순한 구성원을 넘어, 함께 피카디를 만들어가는 파트너로서 일합니다.
- 문제가 보이면 “내가 해결하면 팀에 어떤 도움이 될까?”를 먼저 떠올립니다.
- 타협하지 않는 높은 기준을 가지고 누구에게 보여주어도 자랑스러운 결과물을 만드는 것, 그것이 우리의 자부심입니다.

**끊임없이 배웁니다.**

- 피카디는 ‘내가 맞다’는 확신보다 ‘혹시 내가 틀렸을 수도 있다’는 열린 마음을 더 중요하게 생각합니다.
- 피카디는 이미 알고 있는 방식에 안주하지 않고, 새로운 방식과 관점을 탐색하는 걸 두려워하지 않습니다.
- 불편하더라도, 시간이 걸리더라도, 우리가 진짜 알아야 할 것은 무엇인지 찾아가는 과정을 소중히 여깁니다.

**기술로 임팩트를 만들어냅니다.**

- 피카디는 AI가 사람을 대체하는 게 아니라, 사람의 가능성을 확장시켜준다고 믿습니다.
- 모든 역할에서 AI를 활용할 수 있도록 고민하고, 더 똑똑하게 일할 수 있는 환경을 만들어 갑니다.
- 단순 반복적인 일에서 벗어나, 더 큰 임팩트를 만들고, 더 창의적인 일에 몰입할 수 있도록 돕는 게 우리의 목표입니다.

### Screening interview guideline

#### **process**

- intro → communication → tech/product → ai workflow → q&a/wrapup

#### **how to question & answer**

- 답변은 짧고 간결하게, 1분에서 최대 2분 정도로 커뮤니케이션.
- 주로 팩트 위주의 답변. 느낌, 의견이 있다면 짧게.

#### pre-made questions

#### CTO's hiring standard for screening interview

- 전반적인 커뮤니케이션 흐름.
- 기술 전문성 및 전달 능력.
- 포지션과 팀에 대한 관심도.

**sample technical questions evaluation standard**

- software engineering
	- 개발하면서 제일 힘들었던 상황은 언제였나. 어떻게 해결했나.
	    - 디버깅을 진짜 힘들게 했던 경험.
	- 리팩터링/구조개선/시스템 마이그레이션 등 큰 개선이 필요하다고 느꼈던 경험.
- frontend
    - 상태 관리는 어떤 식으로 했는지.
    - 어떤 어려움이 있었는지.
- backend
    - db
        - 스키마 설계시 어떤 것들을 중점적으로 고려했는지.
        - query monitoring, optimization 경험이 있는지.
    - monitoring
        - 디버깅을 어떤 식으로 했는지.
        - 기억에 남는 장애 경험. 어떤 식으로 픽스했는지.
- ai related
    - 어떤 제품인지.
    - 정확도 개선은 어떻게 했는지.
        - evaluation 방법.
    - ops
        - 동시성에 대한 고민은 어떻게 했는지.
